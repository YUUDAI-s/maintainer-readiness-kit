# Maintainers

## Primary Maintainer

- GitHub: [YUUDAI-s](https://github.com/YUUDAI-s)

## Current Maintainer Responsibilities

- Review incoming issues and classify bugs, feature requests, and documentation
  gaps.
- Review pull requests for scope, tests, documentation impact, and security
  risk.
- Maintain the release workflow, changelog, GitHub Actions smoke checks, and
  repository templates.
- Keep the project dependency-light and suitable for solo and small-team open
  source maintainers.
- Triage security-related reports according to `SECURITY.md`.

## Review Policy

- Changes should stay focused and include tests or a manual verification note.
- Features that introduce network calls must be opt-in and documented.
- The `inspect` command must remain read-only except for the explicitly chosen
  report output path.
- The project should not claim that any repository qualifies for external
  maintainer programs; it only reports evidence.
